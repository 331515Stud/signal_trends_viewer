# Алгоритм обработки и фильтрации данных сигналов

## 1. Общая архитектура

Пайплайн состоит из пяти последовательных этапов:

```
raw.csv → Парсинг → Извлечение признаков → Стандартизация → PCA → HDBSCAN → Фильтрация → raw_filtered.csv
```

---

## 2. Исходные данные

Файл `raw.csv` — CSV с разделителем `;`, содержащий чанки (окна) записанных сигналов.

**Структура каждого чанка:**

| Поле | Описание |
|------|----------|
| `chunkID` | Уникальный идентификатор чанка |
| `timestamp` | Время записи |
| `U_A_rms`, `U_B_rms`, `U_C_rms` | RMS-значения напряжений по трём фазам |
| `I_A_rms`, `I_B_rms`, `I_C_rms` | RMS-значения токов по трём фазам |
| `sample_rate` | Частота дискретизации |
| `signal_length` | Количество отсчётов в чанке |
| `U_A_signal` ... `I_C_signal` | Сигналы в base64 (каждый отсчёт — 3 байта, little-endian, signed int24) |
| `U_mult`, `U_dev`, `I_mult`, `I_dev` | Множители и делители для масштабирования |

**Константы по умолчанию:**

| Параметр | Значение | Описание |
|----------|----------|----------|
| `SAMPLE_RATE` | 25 600 Гц | Частота дискретизации |
| `CHUNK_LEN` | 2048 | Длина чанка (отсчётов) |
| `N_CHANNELS` | 6 | Количество каналов (3 фазы × 2 типа: U и I) |
| `N_FFT_BINS` | 128 | Количество бинов FFT-спектра |
| `FUNDAMENTAL_HZ` | 50.0 Гц | Базовая частота сети |

## 3. Декодирование сигналов

Сигналы хранятся в base64. Каждый отсчёт — 3 байта (24-битное целое число, little-endian, со знаком).

**Алгоритм декодирования (межканальный мультиплекс):**

```
raw_bytes = base64_decode(signal)
n_samples = len(raw_bytes) / (3 × 6)
для каждого канала ch ∈ [0..5]:
    для каждого отсчёта i ∈ [0..n_samples):
        offset = (i × 6 + ch) × 3
        value = int24_from_bytes(raw_bytes[offset:offset+3])
        signal[ch][i] = value / 2²³
```

Декодированные значения нормализуются к диапазону [-1.0, 1.0].

## 4. Извлечение признаков (Feature Extraction)

Из каждого чанка извлекается вектор признаков, состоящий из трёх компонент:

### 4.1. FFT-спектр (128 × 6 = 768 признаков)

- Каждый канал умножается на окно Ханнинга
- Вычисляется FFT → амплитудный спектр
- Берутся первые 128 бинов
- Нормализация: деление на максимум каждого канала

### 4.2. RMS-амплитуды (6 признаков)

```
RMS[ch] = sqrt(mean(signal[ch]²))
```

Одно значение RMS на канал — суммарная мощность сигнала.

### 4.3. THD — Total Harmonic Distortion (6 признаков)

```
THD[ch] = sqrt(Σ amp[ch, f₀×n]², n=2..7) / amp[ch, f₀]
```

Отношение суммарной мощности 2–7 гармоник к мощности основной частоты (50 Гц).

**Итоговый размер вектора raw_vector: 768 + 6 + 6 = 780**

## 5. Оценка качества сигнала (SignalQuality)

Для каждого чанка вычисляются шесть метрик:

| Метрика | Формула / Описание | Хорошо | Плохо |
|---------|---------------------|--------|-------|
| `clipping_ratio` | Доля отсчётов с \|signal\| ≥ 0.99 | < 1% | > 5% |
| `snr_db` | Сигнал/шум в дБ (шум ≈ 20-й перцентиль дифференциалов) | > 40 дБ | < 10 дБ |
| `rms_stability` | Коэффициент вариации RMS по окнам 256 отсчётов | < 0.1 | > 0.5 |
| `thd_max` | Максимальный THD среди всех каналов | < 0.1 | > 0.5 |
| `zero_ratio` | Доля отсчётов с \|signal\| < 1e-10 | < 1% | > 10% |
| `dynamic_range_db` | Разница между max и min RMS среди каналов | < 20 дБ | > 40 дБ |

### Числовой скор качества

```python
score = 1.0

if clipping_ratio > 0.01:
    score *= max(0.0, 1.0 - (clipping_ratio - 0.01) × 10)
if snr_db < 10:
    score *= max(0.1, snr_db / 10)
if rms_stability > 0.5:
    score *= max(0.1, 1.0 - (rms_stability - 0.5))
if thd_max > 0.5:
    score *= max(0.1, 1.0 - (thd_max - 0.5))
if zero_ratio > 0.1:
    score *= max(0.0, 1.0 - (zero_ratio - 0.1) × 5)
```

Итоговый score ∈ [0.0, 1.0]. Штрафы мультипликативны — каждое нарушение усугубляет общую оценку.

## 6. Стандартизация

Z-score нормализация по всем чанкам:

```
flat_vector[i] = (raw_vector[i] - mean) / std
```

Если std < 1e-9, он заменяется на 1.0 (защита от константных каналов).

---

## 7. Снижение размерности (PCA)

- Метод: `sklearn.decomposition.PCA`
- Количество компонент: `N_PCA_COMPONENTS` (по умолчанию 50)
- Ограничение: min(N_PCA_COMPONENTS, число_признаков, число_чанков)
- Автоматически выводится доля объяснённой дисперсии

## 8. Кластеризация (HDBSCAN)

Библиотека: `hdbscan`

| Параметр | Значение по умолчанию | Описание |
|----------|----------------------|----------|
| `min_cluster_size` | 2 | Минимальный размер кластера. **Увеличение → крупнее кластеры, меньше шума** |
| `min_samples` | 1 | Минимальное число точек в окрестности для ядра кластера. **Увеличение → строже отбор в кластер** |
| `metric` | `"manhattan"` | Метрика расстояния. Замена на `"euclidean"` или `"chebyshev"` меняет форму кластеров |
| `cluster_selection_method` | `"eom"` | `"eom"` (Excess of Mass) — гибкие кластеры; `"leaf"` — фиксированные, компактные |
| `cluster_selection_epsilon` | 0.1 | Порог слияния кластеров. **Увеличение → кластеры сливаются** |
| `allow_single_cluster` | False | Разрешить один общий кластер |

### Обнаружение выбросов

Используются outlier scores из HDBSCAN:

1. Нормализация: `scores_norm = (scores - min) / (max - min)`
2. Порог: `percentile = OUTLIER_PERCENTILE` (по умолчанию 97.5%)
3. Всё выше порога → выброс (outlier)

**OUTLIER_PERCENTILE — главный рычаг:**
- **Снижение (напр. 90)** → больше точек считаются выбросами → больше аномалий попадёт в выборку
- **Повышение (напр. 99.5)** → только экстремальные выбросы → фильтрация агрессивнее

## 9. Стратегия отбора representative-записей

### Для кластеров (нормальные точки):

Из каждого кластера выбирается **одна** запись по скору:

```
score = 0.4 × quality_score
       + 0.3 × proximity_score
       + 0.2 × metadata_score
       + 0.1 × diversity_score
```

| Компонента | Диапазон | Описание |
|------------|----------|----------|
| `quality_score` | [0, 1] | Числовой скор из раздела 5.3 |
| `proximity_score` | [0, 1] | Близость к центроиду кластера (нормализованная) |
| `metadata_score` | 0.3 / 0.6 / 1.0 | 1.0 если есть и U и I, 0.6 если одно из двух, 0.3 если нет |
| `diversity_score` | [0, 1] | Средняя дистанция до 5 ближайших соседей / максимальная дистанция |

### Для выбросов (anomalies):

- Если `quality_score ≥ ANOMALY_MIN_QUALITY` → запись **сохраняется**
- Иначе → запись **фильтруется**

## 10. Все настраиваемые параметры

### Глобальные константы

| Параметр | Значение | Где | Эффект изменения |
|----------|----------|-----|-------------------|
| `SAMPLE_RATE` | 25 600 | Константа | Задаёт шкалу частот FFT и расчёт THD |
| `CHUNK_LEN` | 2048 | Константа | Длина окна анализа; уменьшение → худ频率 разрешение |
| `N_FFT_BINS` | 128 | Константа | Сколько бинов спектра берётся; уменьшение → грубее спектр |
| `FUNDAMENTAL_HZ` | 50.0 | Константа | Базовая частота для расчёта THD; изменять при другой сети |

### Параметры пайплайна

| Параметр | CLI-переменная | Значение по умолчанию | Диапазон | Эффект |
|----------|----------------|----------------------|----------|--------|
| **min_cluster_size** | `MIN_CLUSTER_SIZE` | 2 | 2–100+ | Минимальный размер кластера. 2 → мелкие кластеры, много представителей. 50 → только крупные паттерны |
| **n_pca_components** | `N_PCA_COMPONENTS` | 50 | 5–200 | Компонент PCA. Меньше → грубее, но быстрее. Больше → точнее, но шумнее |
| **anomaly_min_quality** | `ANOMALY_MIN_QUALITY` | 0.0 | 0.0–1.0 | Порог качества для аномалий. 0.0 → все аномалии попадают. 0.5 → только качественные |
| **outlier_percentile** | `OUTLIER_PERCENTILE` | 97.5 | 80–99.9 | Перцентиль для выбросов. 90 → много аномалий. 99.5 → только экстремумы |
